# 🛒 Woolworths Store Operations Analytics Dashboard

## 📊 Overview
This project analyzes retail store operations to identify inefficiencies in staffing, customer flow, and operational performance.

The goal is to simulate a real-world analytics use case where store managers can make data-driven decisions to improve efficiency and customer experience.

---

## 🧠 Key Business Questions
- When is the store busiest?
- Are staffing levels aligned with customer demand?
- Do operational delays increase during peak hours?
- Does higher traffic translate into higher revenue?

---

## 🔍 Key Insights
- Peak traffic occurs between **5pm–8pm**, with the highest activity at **8pm**
- Customer-to-staff ratios exceed optimal levels during peak hours, indicating **understaffing risk**
- Trolley collection delays increase during high traffic periods
- Revenue does not scale proportionally with traffic, suggesting **conversion inefficiencies**

---

## 💡 Recommendations
- Reallocate staff to peak evening hours to improve service efficiency
- Implement dynamic staffing strategies based on hourly demand
- Improve operational workflows during high-traffic periods
- Optimize product placement or promotions to improve conversion rates

## 🛠️ Tools & Technologies

## 📁 Dataset
The dataset used in this project is **synthetically generated** to simulate real-world retail store operations.

It includes:
- Customer foot traffic (hourly)
- Staffing levels
- Operational metrics (trolley collection, cleaning)
- Transaction data
